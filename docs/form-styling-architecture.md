
# توثيق آلية عمل نظام تخصيص النماذج

هذا التوثيق يشرح كيفية تخصيص مظهر النماذج في التطبيق، وكيف تتم معالجة التغييرات على عنوان النموذج والخلفية ولون النص وحجمه ومحاذاته.

## آلية عمل النظام والاتصال بين المكونات

### 1. تخزين وإدارة البيانات
- **useFormStore.ts**: المخزن المركزي لجميع بيانات النموذج باستخدام مكتبة zustand
  - يخزن `formState` الذي يحتوي على بيانات النموذج
  - يخزن `style` الذي يحتوي على أنماط النموذج (الألوان، الحجم، التقويس)
  - يخزن `floatingButton` إعدادات الزر العائم
  - يوفر دوال لتحديث هذه البيانات مثل `setFormState`, `updateFloatingButton`

### 2. محررات التخصيص
- **FormTitleEditor.tsx**: واجهة المستخدم للتحكم في عنوان النموذج
  - يستقبل `formTitleField` كـ prop ويرسل التغييرات عبر `onUpdateTitleField`
  - يتيح تحرير لون النص، لون الخلفية، الحجم، والمحاذاة
  - **النقطة المهمة**: يستخدم `handleUpdateStyle` لتحديث خصائص النمط

### 3. المعاينة المباشرة
- **FormPreviewPanel.tsx**: حاوية لعرض المعاينة المباشرة
  - يستقبل الـ props من الصفحة الرئيسية ويمررها إلى `FormPreview`
  - يستقبل `refreshKey` لإعادة تحميل المعاينة عند التغيير

- **FormPreview.tsx**: مكون المعاينة المباشرة الرئيسي
  - يستقبل جميع البيانات والأنماط
  - ينظم عرض الحقول بالترتيب الصحيح
  - يضيف عنوان النموذج وزر الإرسال تلقائياً إذا لم يكونا موجودين

### 4. مكونات عرض الحقول
- **FormField.tsx**: يحدد نوع الحقل المناسب ويختار المكون المناسب لعرضه
  - يستقبل `field` و `formStyle` ويمررهما للمكون المناسب

- **TitleField.tsx**: مكون عرض عنوان النموذج
  - يستقبل `field` و `formStyle`
  - يطبق جميع الأنماط المحددة (لون النص، لون الخلفية، الحجم، المحاذاة)
  - **مهم**: يستخدم قيم ثابتة بوحدة البكسل بدلاً من الـ rem لضمان عرض متناسق

## النقاط الحاسمة لحل المشكلات السابقة

### 1. استخدام وحدات البكسل بدلاً من Rem

**المشكلة السابقة**: كان استخدام وحدات rem للخطوط يتسبب في مشكلات العرض عبر البيئات المختلفة.

**الحل**: 
- في ملف `TitleField.tsx`، تم تعديل الكود لاستخدام قيم بكسل ثابتة:
```typescript
// استخدام قيم بكسل ثابتة بدلاً من rem
const fontSize = isFormTitle ? '24px' : '20px'; // 1.5rem = 24px, 1.25rem = 20px
const descriptionFontSize = '14px'; // 0.875rem = 14px
```

### 2. التحكم المباشر بخلفية العنوان

**المشكلة السابقة**: لم يكن تغيير لون خلفية العنوان ينعكس بشكل صحيح.

**الحل**:
- في `FormTitleEditor.tsx` إضافة تحكم خاص للون الخلفية:
```typescript
const [backgroundColor, setBackgroundColor] = useState(formTitleField?.style?.backgroundColor || '#9b87f5');

// دالة التحديث
const handleUpdateStyle = (property: string, value: string) => {
  if (property === 'backgroundColor') setBackgroundColor(value);
  // ...باقي الكود
}
```

- في `TitleField.tsx` التطبيق المباشر للون الخلفية:
```typescript
const backgroundStyle = {
  backgroundColor: backgroundColor,
  padding: '16px',
  borderRadius: formStyle.borderRadius || '8px',
  width: '100%',
  boxSizing: 'border-box' as BoxSizing,
};
```

### 3. استخدام مفتاح تحديث للمعاينة

**المشكلة السابقة**: لم تكن التغييرات تظهر مباشرة في المعاينة.

**الحل**:
- في `FormBuilderEditor.tsx`، استخدام مفتاح `refreshKey` لإعادة رسم المعاينة:
```typescript
const [previewRefresh, setPreviewRefresh] = useState(0);

// عند تغيير أي إعداد
setPreviewRefresh(prev => prev + 1);

// استخدام المفتاح في المعاينة
<FormPreviewPanel 
  key={previewRefresh}
  // ... باقي الخصائص
/>
```

### 4. تثبيت قيم الخصائص للوصف

**المشكلة السابقة**: كانت هناك مشكلة في خصائص وصف العنوان.

**الحل**:
- استخدام قيمة ثابتة لوزن الخط في الوصف:
```typescript
// استخدام قيمة ثابتة 'normal' بدلاً من متغير
const descriptionStyle = {
  fontWeight: 'normal', // قيمة ثابتة بدلاً من متغير descriptionFontWeight
  // ...باقي الخصائص
};
```

## مثال لتدفق البيانات من المحرر إلى المعاينة

1. المستخدم يغير لون الخلفية في `FormTitleEditor.tsx`
2. يتم استدعاء `handleUpdateStyle('backgroundColor', newColor)`
3. يتم تحديث `formTitleField.style.backgroundColor` في مخزن الحالة
4. يتم زيادة `previewRefresh` بمقدار 1 لإعادة رسم المعاينة
5. يتم إعادة رسم `FormPreview.tsx` مع القيم الجديدة (بسبب تغير الـ key)
6. `FormPreview.tsx` يمرر البيانات إلى `FormField.tsx`
7. `FormField.tsx` يحدد أن الحقل من نوع 'form-title' ويستخدم `TitleField.tsx`
8. `TitleField.tsx` يطبق `backgroundColor` الجديد على العنصر

## اتصال المعاينة بالمتجر

لضمان تطبيق نفس المظهر في المتجر، تم اتباع الخطوات التالية:

1. تعريف أنماط CSS متسقة في `codform_form.liquid`
2. استخدام المتغيرات CSS مثل `--form-primary-color`
3. استخدام نفس أسماء الفئات مثل `codform-title`, `codform-form-title`
4. تضمين خصائص `!important` في حالات معينة لتجاوز أنماط المتجر الأخرى

## التوافقية مع اللغة العربية

- استخدام خاصية `dir="rtl"` عندما تكون اللغة هي العربية
- ضبط المحاذاة النصية بناءً على اللغة (يمين للعربية، يسار للإنجليزية)
- تطبيق الهوامش والحشوات بشكل متناسق مع اتجاه النص

## التوصيات للمحافظة على عمل النظام

1. الحفاظ على استخدام قيم البكسل الثابتة في `TitleField.tsx` عند تحديد أحجام الخطوط
2. عدم تغيير آلية تحديث المعاينة باستخدام `refreshKey`
3. الحفاظ على الفصل الواضح بين مخزن الحالة (`useFormStore.ts`) ومكونات العرض
4. عند إجراء أي تعديلات، اختبارها على كل من المعاينة والمتجر للتأكد من تناسق المظهر
5. الاهتمام الخاص بتوافقية النظام مع اللغتين العربية والإنجليزية
