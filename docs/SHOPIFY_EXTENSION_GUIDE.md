
# دليل امتدادات Shopify

هذا الدليل يوثق عملية تطوير ونشر امتدادات Shopify للنماذج.

## نظرة عامة على امتدادات Shopify

امتدادات Shopify تسمح بإضافة وظائف مخصصة إلى متاجر Shopify. بالنسبة لنا، نستخدم امتدادات السمات (Theme Extensions) لعرض نماذج الدفع عند الاستلام في المتجر.

## هيكل امتداد السمة

مسار الامتداد: `extensions/theme-extension-codform/`

الملفات الرئيسية:
- `shopify.extension.toml`: ملف تكوين الامتداد
- `blocks/codform_form.liquid`: قالب عرض النموذج في المتجر
- `assets/`: مجلد للأصول مثل CSS وJavaScript

### ملف التكوين (shopify.extension.toml)

```toml
name = "CODFORM - نماذج الدفع عند الاستلام"
type = "theme"
handle = "theme-extension-codform"

[[metafields]]
namespace = "codform"
key = "settings"

[settings]
fields = []
```

### قالب النموذج (codform_form.liquid)

القالب يتكون من:
1. إعدادات القالب (schema)
2. كود HTML لعرض النموذج
3. نص JavaScript لتحميل وعرض النموذج بشكل ديناميكي

## عملية نشر امتداد Shopify

### 1. التحقق من صحة الامتداد

قبل النشر، يجب التحقق من صحة ملفات الامتداد باستخدام:

```bash
./deploy-shopify.js validate
```

التحقق يشمل:
- صحة ملف `shopify.extension.toml`
- صحة قوالب Liquid
- التحقق من تنسيق الملفات

### 2. نشر الامتداد

لنشر الامتداد:

```bash
./deploy-shopify.js deploy-extensions
```

هذا الأمر:
- يقوم بضغط ملفات الامتداد
- رفعها إلى Shopify Partners
- الحصول على رابط التثبيت

### 3. تثبيت الامتداد في المتجر

بعد النشر، يمكن تثبيت الامتداد في المتجر باستخدام رابط التثبيت.

## إرشادات مهمة لتطوير امتدادات Shopify

### 1. ملف التكوين (shopify.extension.toml)

- يجب أن يحتوي على اسم ونوع وmesؤول صالحين
- `name` يجب أن يصف وظيفة الامتداد بوضوح
- `type` يجب أن يكون "theme"
- `handle` يجب أن يكون فريداً ومتسقاً

### 2. قوالب Liquid

- يجب أن تتبع تنسيق Shopify Liquid الصحيح
- يجب أن تتضمن schema صالحة لإعدادات القالب
- تجنب استخدام CSS أو JavaScript مضمن قدر الإمكان
- استخدم الأصول الخارجية لتحسين الأداء والصيانة

### 3. حدود وقيود Shopify

- حجم الامتداد: يجب ألا يتجاوز 10 ميجابايت
- عدد الملفات: محدود (يفضل تقليل عدد الملفات)
- الوصول: الامتدادات لها وصول محدود إلى API الخاص بـ Shopify

### 4. إرشادات RTL واللغة العربية

- تأكد من إضافة الاتجاه المناسب (`dir="rtl"`) للعناصر العربية
- استخدم CSS متوافق مع RTL لضمان العرض الصحيح
- اختبر الامتداد في متاجر تستخدم اللغة العربية

## مشاكل شائعة وحلولها

### 1. فشل التحقق من الصحة

**المشكلة**: فشل أمر `validate` مع أخطاء في ملف التكوين.

**الحل**:
- تحقق من تنسيق ملف TOML
- تأكد من أن جميع الحقول المطلوبة موجودة
- تأكد من عدم وجود أحرف خاصة غير مدعومة

### 2. فشل النشر

**المشكلة**: فشل أمر `deploy-extensions`.

**الحلول**:
- تأكد من الاتصال بحساب Shopify Partners
- تحقق من أن الإصدار الجديد له رقم إصدار مختلف
- تحقق من حجم الامتداد (يجب أن يكون أقل من 10 ميجابايت)

### 3. النموذج لا يظهر في المتجر

**المشكلة**: الامتداد مثبت ولكن النموذج لا يظهر.

**الحلول**:
- تأكد من وضع Block في الموقع الصحيح في قالب السمة
- تحقق من وجود نموذج مزامن مع المتجر
- تحقق من سجلات الأخطاء في المتصفح

## أفضل الممارسات لتطوير امتدادات Shopify

1. **اختبر دائمًا قبل النشر** - استخدم `validate` دائمًا
2. **احتفظ بتنسيق متسق** في ملفات التكوين والقوالب
3. **وثق التغييرات** لكل إصدار جديد
4. **قم بتحسين الأداء** عن طريق تقليل الطلبات والحجم
5. **اختبر في عدة متصفحات وأجهزة** للتوافق الأقصى
6. **احتفظ بنسخة احتياطية** قبل النشر الجديد

## نصائح للتصحيح

- استخدم Console Browser لتتبع أخطاء JavaScript
- تحقق من الطلبات الشبكية للتأكد من تحميل النماذج
- استخدم وضع التفتيش لمعرفة ما إذا كان عنصر النموذج موجودًا في DOM

## قالب نموذجي لـ codform_form.liquid

```liquid
{% comment %}
  CODFORM - نموذج الدفع عند الاستلام
  هذا المكون يقوم بعرض نموذج مخصص من خدمة CODFORM
{% endcomment %}

{% schema %}
{
  "name": "نموذج CODFORM",
  "target": "section",
  "settings": [
    {
      "type": "text",
      "id": "form_id",
      "label": "معرف النموذج",
      "info": "أدخل معرف النموذج من لوحة تحكم CODFORM"
    }
  ]
}
{% endschema %}

<div id="codform-container" class="codform-wrapper" data-form-id="{{ block.settings.form_id }}">
  <div class="codform-loading">جاري تحميل النموذج...</div>
</div>

<script>
  (function() {
    // كود JavaScript لتحميل النموذج
  })();
</script>
```

## نموذج لسير العمل في تطوير امتداد جديد

1. **قم بإنشاء مجلد للامتداد** ضمن مجلد `extensions/`
2. **أضف ملف التكوين** `shopify.extension.toml`
3. **قم بإنشاء مجلد `blocks`** وأضف قوالب Liquid
4. **اختبر محليًا** باستخدام أمر `validate`
5. **قم بالنشر** باستخدام أمر `deploy-extensions`
6. **اختبر في المتجر** بعد التثبيت
7. **وثق التغييرات والميزات**
